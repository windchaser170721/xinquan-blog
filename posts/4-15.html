H-WM ​

核心

H-WM 如何做到把符号推理的长程稳定性和视觉表征的感知 grounding结合起来？本质是“两层时间尺度 + 两层世界模型”

逻辑世界模型本身，它本质上是一个经过 CoT 轨迹微调的 LLM 版符号转移模型。作者先把每个子任务标注成逻辑状态、逻辑动作、观测和机器人构型，再把 episode 里的 action/state transition 转成 chain-of-thought 解释，去微调一个 base LLM。推理时，这个逻辑模型既当 search model，又当 evaluator：一方面提出候选逻辑动作和下一状态，另一方面对部分轨迹按逻辑一致性和目标匹配程度打分。也就是说，它不是传统意义上只“生成”的 world model，而更像一个兼具符号动力学模型 + 结构化 reward / heuristic 的高层规划器。论文实现上用的是 Qwen3-4B-Instruct，加 LoRA 微调 10k steps。

视觉世界模型则比很多视频生成式 world model 更“克制”。把…编进一个 understanding expert，再由 prediction expert 通过迭代去噪预测一个 latent subgoal feature，并用 frozen ve 提取的目标图像特征做监督，损失函数用的是 sliced Wasserstein loss。设计思想：对 VLA 来说，真正有用的不是一张看起来逼真的未来帧，而是一个足够稳定、足够压缩、又能表达“下一子任务完成态”的目标表征。

下游控制策略: guided VLA，分 understanding 、goal 、 action三个专家部分：ue 处理当前高频观测和 logical action，ge 处理视觉世界模型预测出的 latent goal，ae 通过 cross-attention 同时看两边，再结合当前 joint configuration 生成动作 chunk。特意限制了反向信息流，只允许 ae attend to 另外两个 expert，以保持层级结构稳定。整个 policy 用 fm objective 训练，初始化来自 PaliGemma；视觉编码器用的是 SigLIP。加了一个 subtask completion predictor，在每一步判断当前子任务是否完成，从而决定何时切换到下一逻辑子任务。没有这个 completion head，分层框架很容易在“什么时候切子任务”上失稳。

贡献：把“世界模型”从单一像素预测，重新定义成了一个层级中间变量系统。高层变量是 symbolic transition，低层变量是 visual latent subgoal，中间再通过 completion prediction 和 guided VLA 把规划和执行闭起来。严格说更像一个神经符号式 long-horizon VLA guidance framework，而不是那种统一地从历史观测直接 rollout future tokens 的 monolithic world model。这也是它效果好的原因：它没有试图让一个模型同时承担所有难题，而是把任务结构、感知 grounding、动作生成分别放在最合适的层级上。

局限：第一，依赖显式逻辑状态表示；LIBERO 里作者是先用预设计 predicate classifier 回放打标签，再人工筛错，实际代价不低。第二，它对任务可分解性有隐含假设，前提是任务能被组织成清晰的 symbolic subtasks。第三，它虽然比视频生成 world model 更稳，但系统复杂度也更高：逻辑模型、视觉模型、guided policy、completion head 都要协调。论文自己也承认训练成本和对结构化 logical state 的依赖，是目前的主要限制。

收获：第一，长程问题不要直接压给 low-level policy，而是给它喂慢频率、结构化的中间变量。第二，symbolic guidance 和 visual guidance 不是二选一，而是前者负责“做什么”，后者负责“做到什么样才算完成”。第三，latent subgoal 比 pixel future 更适合当执行条件，这点对你现在关心的 hierarchical VLA、atomic skill、甚至 teacher-student distillation 都很有启发。

DreamZero ​

核心

机器人策略不该只学“看见什么就输出什么动作”的 VLA 映射，而应该同时学会未来世界会怎么变化

联合预测未来视频和对应动作能把动作学习从“纯模仿”转成“面向未来视觉结果的逆动力学”，从而更强地泛化到新环境、新动作甚至新机器人。VLA 往往知道“要把东西放到哪里”，但一旦遇到训练里没出现过的动作模式，比如更复杂的接触、操作顺序、空间轨迹变化，就容易退化成自己最熟悉的 pick-and-place。DreamZero 的切入点是：视频扩散模型在大规模视频上已经学到很强的时空先验，那就别只拿它当感知 backbone，而是直接让它承担“世界如何演化”的建模职责，再把动作预测绑到这个未来演化过程上。

结构：它把视觉上下文先编码成 VAE latent，把语言指令送进文本编码器，把 proprio state 送进状态编码器，然后交给一个 autoregressive DiT 主干，用 flow matching 同时生成未来视频帧和动作。论文特别强调两点。第一，它不是分开做“视频预测模型 + 逆动力学模型”，而是一个模型端到端联合预测，目的就是让视频未来和动作之间强对齐。第二，它选的是 autoregressive 而不是 bidirectional，因为这样能用 KV cache、保留原始 FPS 的时序对齐，而且每执行完一个 action chunk，就把真实观测重新写回 cache，避免纯 rollout 的误差越滚越大。

VLA 更像直接学 observation→action，DreamZero 更像先内隐地想象“接下来世界会变成什么样”，再从这个未来去反推动作。 这也是论文里反复强调的“joint video-action prediction”与“inverse dynamics from predicted futures”。这套设计还有一个很重要的附带收益：它更适合吃异构、非重复的数据，不必每个任务都反复收很多标准演示。

实验：作者在 AgiBot G1 上收了约 500 小时、7.2K episode、22 个环境的数据，明确强调数据收集哲学是优先追求任务和场景广度，而不是同一任务的重复演示。实验结果方面首先是unseen tasks的提升，很多 pretrained VLA 在 unseen task 上虽然偶尔也会去抓东西，但常常只是把熟悉的抓放模板套上去；DreamZero 则更像真的在做视觉层面的 future planning，所以动作和视频预测之间的一致性更强。另一个很亮眼的点，是它做了两种 cross-embodiment transfer。第一种是只给视频、不给动作标签：来自另一个机器人 YAM 或者人类第一视角的 10–20 分钟视频演示，就能把 AgiBot G1 在 unseen tasks 上的 task progress 从 38.3% 提到 54.3% / 55.4%。第二种更激进：把 AgiBot 预训练好的 DreamZero 用在新机器人 YAM 上，只用 30 分钟、55 条 trajectory 的新机体 play data 做后训练，就能保留零样本泛化，去操作新物体。它在暗示一个很强的方向：世界模型学到的“物理变化先验”也许比直接学 policy 更容易跨 embodiment 迁移。

Motus ​

核心

把 VLA、world model、inverse dynamics、video generation 和 video-action joint prediction 放进同一个生成框架（latent action world model）里

结构：结构层面的统一：作者提出了一个 Mixture-of-Transformers (MoT)，三个 expert 分别负责视觉语言理解、视频生成/世界建模和动作生成；三者通过共享的 Tri-model Joint Attention 互相通信。生成 expert 建在 Wan 2.2 5B 上，理解 expert 建在 Qwen3-VL-2B 上，动作 expert 则和视频 expert 保持相同深度，再配一个 UniDiffuser-style scheduler，给视频和动作分配不同的时间步和噪声尺度，这样一个模型就能切换成不同模式：VLA 模式、WM 模式、IDM 模式、VGM 模式，以及视频-动作联合预测模式。

结构：latent action：作者没有直接把不同机器人的显式 action 硬对齐，因为跨 embodiment 的动作空间本来就不一致；他们改成从光流里学动作表征，把 optical flow (连续两帧之间，每个像素看起来往哪儿移动了多少)当成一种像素级的 “delta action”。具体做法是先用 DPFlow 提取相邻帧光流，再用一个 DC-AE 压缩成低维 latent，论文里写的是压成 4 个 512 维 token。为了让这个 latent space 不只学到“视觉运动”而脱离真实控制，他们又混入少量有 action 标签的数据和 task-agnostic (沿用 AnyPos 的思路，用 Curobo 以 task-agnostic 的方式随机采样目标机器人的 action space，收集 image-action pairs。这类数据教它“对于这台机器人，哪些动作在物理上可行，会导致什么基本视觉变化”。AnyPos 的解释：task-agnostic data 关注的是覆盖 embodiment 的可行动作空间，而不是围绕某个具体任务收集长序列演示。)数据做弱监督，（训练时大约是 90% 无标签重建 + 10% 有标签轨迹/任务无关动作监督）。这一步让 action expert 能在大量无动作标签的视频上做预训练，而不是只能吃机器人演示。

训练recipe：一个三阶段 + 六层数据金字塔。六层数据从下往上大致是：web 级预训练基础模型、第一视角人类视频、合成数据、task-agnostic image-action 数据、多机器人任务轨迹，以及目标机器人自己的轨迹数据；数量越往下越大，质量和目标相关性越往上越高。三阶段分别是：第一阶段先把视频生成/world model 能力练起来；第二阶段把三个 expert 和 latent actions 一起预训练；第三阶段再用目标机器人数据做 SFT，把通用先验收敛到具体 embodiment 上。Motus 先吸收通用视频/理解先验，再把动作知识通过 latent action 接进来。

实验上：想证明“统一模型真的能在多个角色上都工作”。论文报告说，在 RoboTwin 2.0 仿真里，Motus 相比 X-VLA 有大约 +15% 的提升，相比 π0.5 有大约 +45% 的提升；在 LIBERO-Long 上，Motus 的平均分是 97.6，和 X-VLA 持平达到 SOTA；在 RoboTwin 2.0 randomized 设定下，Motus 作为 VLA 模式 的平均成功率是 83.90，作为视频-动作联合预测模式是 87.02；作为 IDM 模式 时，它的 action MSE 是 0.014，优于文中的 ResNet18+MLP 和 DINOv2+MLP 基线；作为 world model 模式 时，在两种真实机器人平台上的 FID/FVD/SSIM/LPIPS/PSNR 也给了完整结果。也就是说，作者确实不是只拿一个“统一框架”口号来做 policy，而是尝试证明同一套模型在多个 embodied 建模任务上都能用。

DreamZero 的重点是“视频世界模型直接变成策略主体”，强调未来视频和动作的联合生成；H-WM 的重点是“分层中间变量”，高层逻辑世界模型做符号推进，低层视觉世界模型提供 latent subgoal；而 Motus 更像是在追求建模范式的一体化：它不一定在层级规划上最强，也不把视频未来当作唯一中心，而是想用一个统一生成框架把理解、世界建模、反向动力学和动作输出全部收进去。Motus 的“统一”更偏系统和分布层面的统一，而不是 H-WM 那种“按时间尺度分层”的统一。这个判断来自它明确要统一的五个分布、三 expert 结构，以及 UniDiffuser 式的 mode switching。

评价：机器人 foundation model 不该只统一“输入输出接口”，还要统一“生成任务类型”。同一个模型既能看懂、又能想象、还能反推动作和直接控动作。它的强项是把预训练 VLM 的语义先验、预训练视频模型的时空先验和latent action 的跨 embodiment 运动先验绑在一起；代价也很明显：系统复杂，显式模块多，latent action 仍然需要单独的光流-VAE 学习和三阶段训练，所以它更像一个“统一大框架”，而不是一个极简、端到端、轻量化的 low-level policy 方案。
