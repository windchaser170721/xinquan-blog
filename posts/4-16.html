VGGT ​

核心

把相机、深度、点图、跟踪特征统一进一个大 Transformer 里，直接多视图前向推理，尽量绕开传统 SfM/BA 那套慢速后处理

输入：是一组观察同一场景的 RGB 图像，输出则是每张图对应的相机参数、深度图、point map 和跟踪特征。这里的 point map 是“每个像素对应一个 3D 点”；这些 3D 点都定义在第一张图相机坐标系里，所以它是一个统一参考系下的几何表达。论文还特别说明，输入图像的顺序除了第一帧作为参考系外，其余是任意的，网络对其余帧保持 permutation equivariant。

创新：是把一个通用大 Transformer 真正做成了多视图几何统一器。先用 DINOv2 把每张图 patchify 成 token，再给每帧额外加一个 camera token，随后在主干里交替使用 frame-wise self-attention 和 global self-attention。前者在单帧内部聚合，后者跨帧聚合，这样既保留每张图自身结构，又让多视图之间充分交换信息。作者默认用了 24 层这样的交替注意力结构，并强调主干里没有 cross-attention。

输出头：相机参数由 camera token 经过额外 self-attention 和线性层得到；深度图、point map 和 tracking feature 则由图像 token 经过 DPT head 上采样得到。对点跟踪，VGGT 本体并不是直接输出每个点的轨迹，而是先输出 dense feature，再交给一个 CoTracker2 风格的 tracking module 去根据查询点做跨帧关联；这两个模块是联合端到端训练的。也就是说，VGGT 不是简单把“深度估计器+位姿回归器+跟踪器”拼在一起，而是在一个共享 backbone 上学统一几何表征。注意：在 VGGT 里，真正跨帧推理、几何对齐主要发生在 backbone；而 DPT head 更像是每帧独立的 dense decoder。补充材料也明确说了：跨帧关系只在 feature backbone 中处理，DPT heads 是对每一帧独立做预测；并且一个 DPT head 平均每帧大约增加 0.03 秒推理时间和 0.2 GB 显存。

over-complete predictions：按几何关系说，很多量其实互相可推，比如 point map 可以反推相机，point map 配合相机也能推出深度。但发现：训练时显式同时预测这些冗余但相关的量，性能反而更强；而在测试时，拿“独立预测的深度 + 相机”去重建点云，往往比直接用 point map head 还更准。可以理解为一种“多任务几何一致性监督”：虽然输出之间有闭式关系，但同时学它们，能逼 backbone 学到更稳的 3D 结构。

训练：VGGT 是个重模型。它大约 1.2B 参数，总损失是相机、深度、point map、tracking 的多任务和，其中 tracking loss 权重设为 0.05。训练时每个 batch 从一个场景里随机抽 2 到 24 帧，输入最长边缩放到 518，使用大量 3D 标注数据联合训练，包括 Co3Dv2、BlendMVS、DL3DV、MegaDepth、Kubric、ScanNet、HyperSim、Mapillary、Habitat、Replica、Virtual KITTI、Aria Synthetic Environments / Digital Twin 等，并在 64 张 A100 上训练 9 天。

实验：相机位姿估计上，在 RealEstate10K 和 CO3Dv2 的 10-frame 设置里，VGGT 的纯前向版本 AUC@30 分别做到 85.3 / 88.2，比 DUSt3R、MASt3R、VGGSfM v2 以及同期的 Fast3R、FLARE 等都高；而推理时间只有 约 0.2 秒/scene（H100）。如果再加 BA，指标还能进一步到 93.5 / 91.8，同时速度仍然比传统优化流水线快很多。

多视图深度和点图重建：在 DTU 上，它在不知道测试时 GT 相机的条件下，把 Overall 从 DUSt3R 的 1.741 降到 0.382，已经接近那些测试时拿到真相机参数的方法；在 ETH3D 上，它的 Depth + Cam 方案比直接 point map head 更好，Overall 从 0.709 进一步到 0.677，再次印证了“深度 + 相机组合优于直接点图头”的观察。

意义：以前高质量多视图几何必须依赖显式优化，神经网络最多当特征提取器或局部模块；VGGT 证明了 scaling law：一个足够大的、训练充分的前馈模型，已经能直接给出可用级别的相机和几何，而且还能作为下游 backbone。论文里还展示了它对动态点跟踪和 feed-forward novel view synthesis 的促进作用，例如把预训练 VGGT backbone 接到 CoTracker 上后，TAP-Vid RGB-S 的 从 78.9 提升到 84.0。

缺陷：第一，作者自己就承认 BA 后处理仍然还能继续提升结果，说明纯前馈还没完全替代几何优化。第二，虽然推理快，但模型本身很大，训练成本也很高。第三，它主要建模的是静态场景几何；动态视频跟踪那部分更多是把它当强 backbone，而不是说它天生就解决了时序动态重建。第四，它的全局注意力在大规模多图输入下会有明显内存压力，补充材料也提到可以借鉴 LLM 里的效率技术进一步优化。谁能验证动态时序建模的scaling law就nb了。

StreamVGGT ​

核心

视频一帧一帧进来时，边看边做 3D/4D 几何感知，尽量做到低延迟、可交互

解决痛点：像 VGGT 这种强多视图几何模型，本质上依赖 dense-view / global attention，精度高，但天然更适合“把一段序列全拿到手再统一算”；一旦进入在线场景，每来一帧都把整个历史重算一遍，延迟和算力都不现实。StreamVGGT 的思路是借鉴自回归大语言模型：把视频帧看成按时间到来的 token 序列，用 temporal causal attention 只看当前帧和过去帧，再把历史 key/value 缓存起来，当作隐式记忆去复用。这样它就能增量式做重建，而不是反复回头全量重跑。

结构：它基本继承了 VGGT 的“几何 foundation model”路线，但把核心交互方式改成了因果式。模型由 image encoder、spatio-temporal decoder、multi-task heads 三部分组成。输入图像先被 DINO 编成 patch tokens；然后在解码器里，不再做全局 all-to-all attention，而是把原本的 global self-attention 改成时间因果注意力，让每个 token 只能访问当前和过去帧。训练时仍然可以把整段序列一起送入，但通过 causal mask 强制模型学会“只依赖过去”；推理时则把历史帧的 key/value 缓存下来，对当前帧做 cross-attention，从而复现训练时的因果行为。

输出：每帧都会预测相机参数、point map、depth map、point tracks。论文还提到用一个可学习的 camera token 把第一帧标成全局参考系， 于是后续帧都会逐步对齐到这个共享坐标系里。它说明 StreamVGGT 不是只做逐帧单目深度，而是在做一个有全局坐标一致性的流式几何建图。geometry head 还会输出深度和点图的 confidence，以及 dense tracking features，再交给 tracking 分支。

trick：用 VGGT 做 teacher 来蒸馏训练 StreamVGGT。直接让双向、离线、精度高的 VGGT 输出 pseudo-label，给学生模型提供相机、深度、point map、tracks 的软监督。好处：一是统一了多任务监督来源，二是把 VGGT 学到的多视图几何先验直接迁移过来，让流式模型用更少算力接近 teacher 水平。实验里也确实显示，不做 distillation 时性能明显掉，而加了 KD 后会显著回升。

实验：它在 streaming 设定下把质量和速度拉到了一个非常实用的平衡点。在 7-Scenes 和 NRGBD 的 3D reconstruction 上，StreamVGGT 明显优于当时的强 streaming 基线 CUT3R；例如在 ETH3D 上，它的 overall 指标是 0.577，优于 VGGT 的 0.686、MASt3R 的 0.826 和 CUT3R 的 1.411。在动态场景 TUM-dynamics 上，它也比 CUT3R 更接近离线 VGGT，表现出比较强的 4D 重建能力。

深度估计：单帧 depth 上，StreamVGGT 在 Sintel、Bonn、KITTI、NYU-v2 四个数据集上都超过了 CUT3R 和 Point3R，甚至在多项指标上逼近或超过离线 VGGT；比如 Sintel 的 Abs Rel 从 CUT3R 的 0.428 降到 0.254，Bonn 从 0.063 降到 0.052，NYU-v2 从 0.086 降到 0.055。视频 depth 上，它在 Sintel 和 Bonn 也明显强于 CUT3R，只是在 KITTI 上仍然落后于离线 VGGT。也就是说，它的价值不是完全取代离线大模型，而是在在线约束下把深度质量拉得足够高。

相机位姿：它在 ScanNet 和 TUM-dynamics 上也很强。比如 ATE 指标上，ScanNet 从 CUT3R 的 0.099 降到 0.048，TUM-dynamics 从 0.046 降到 0.026；不过在 Sintel 上它没有超过 CUT3R，ATE 是 0.219，略高于 CUT3R 的 0.213。这点很值得注意：StreamVGGT 不是“全榜单无死角第一”，而是总体很强，但在某些动态/复杂外观场景上仍存在和最佳方法互有胜负的情况。

效率：关键是 cache + FlashAttention-2。在在线设定下，引入 FlashAttention 和 cached memory token 后，某个消融里的推理时间从 1135.860 ms 降到 88.193 ms，峰值显存从 5.4 GB 降到 2.7 GB。同时，full-sequence causal inference 和 cached-token streaming inference 在 ETH3D 上给出了相同的 Acc/Comp/Overall（0.782 / 0.723 / 0.753），但最后一帧延迟从 4.7 s 降到 0.07 s。这说明它的缓存机制基本保住了效果。

限制：它虽然是 streaming，但不是严格 O(1) 常数成本。因为它缓存的是历史 KV，序列越长，缓存还是会变大。（后期太多memory优化）论文自己的表 7 和表 8 已经暴露了这一点：40 帧时，StreamVGGT 当前帧推理已到 216 ms / 6.6 GB；200 帧时如果不裁剪缓存，会到 733 ms / 25.3 GB。作者因此又实验了 window-based 和 K-nearest-frames 这类 pruning 策略来压成本。也正因为这个问题，后面才会出现你搜索结果里能看到的 OVGGT、XStreamVGGT，专门去解决“长时视频下缓存无限长”的问题。

概括：VGGT 证明了大一统几何 Transformer 可以做得很强，StreamVGGT 则证明这条路可以进一步改造成“像 LLM 一样按时间流式工作”的在线系统。 它把“离线全局几何推理”改造成了“因果建模 + 历史缓存 + 蒸馏迁移”的在线几何推理范式。

cut3r ​

核心

把 3D 感知从“看完一组图再统一求解”改成“边看边更新、一直记着场景”的过程。

它维护一个会持续演化的内部状态；每来一张新图，这个状态就被更新一次，同时模型立即输出该视角下的相机参数和稠密 3D 几何。这样得到的每帧 point map 都落在同一个坐标系里，可以不断累积成在线更新的场景重建。

解决问题：传统或近年的 3D 方法，要么偏向离线多视图重建，要么需要全局对齐/优化，要么更擅长固定长度输入，而不太适合真正的连续感知流程。作者希望模型既能处理视频流，也能处理稀疏照片集合；既能处理静态场景，也尽量兼容动态内容；而且不只是重建已经看见的部分，还能基于内部状态去“查询”一个没出现过的视角，推断未观测区域的 3D 结构。

persistent state：作者把这个状态也表示成一组 token。在还没看到任何图像之前，这些 state tokens 是一组共享的可学习 token；当第 t 张图进来后，图像先被共享权重的 ViT 编码成 image tokens，再和 state tokens 发生双向交互：一部分交互负责 state update，把当前图像的信息写进状态；另一部分负责 state readout，从状态里读出过去积累的上下文，补充到当前图像的表示里。这个过程由两个相互连接的 transformer decoder 完成。论文里还引入了一个可学习的 pose token，专门抽取和整张图对应的场景级信息，比如相机位姿相关信息。

输出：可以理解成“当前图像的 3D 解释”。第一类是 self frame 下的 point map，也就是每个像素在当前相机坐标系下对应的 3D 点；第二类是 world frame 下的 point map，也就是每个像素在统一世界坐标系中的 3D 点；第三类是从当前帧到世界坐标系的 6-DoF pose。这里论文把世界坐标系定义成第一张图的坐标系。之所以同时预测 self-frame point map、world-frame point map 和 pose，看起来有点冗余，但作者明确说这样更利于训练，因为每个输出都能拿到更直接的监督，还方便利用只有部分标注的数据。

trick：不只是“持续重建”，而是用状态去查询未见视角。作者引入了一个叫 ray map 的表示，把一个虚拟相机的内外参编码成一个 6 通道图像，每个像素存射线的起点和方向。然后用单独的 transformer 编码这个 ray map，让它和当前 persistent state 做交互；但这一步只有 readout，不更新状态，因为它只是查询，不是引入新的观测。通过这种方式，模型可以直接输出这个虚拟视角下的 point map，甚至颜色。作者把它类比成一种“场景级别的补全”：不是补图像 patch，而是利用已经积累的 3D 场景记忆，补一个整视角的几何。

总结：它在脑子里一直维护一个“场景隐式记忆”。新图来了就把记忆修正一下，当前要输出哪个视角的 3D，就从这个记忆里读。这个思路和很多机器人系统里的“地图/记忆体”很像，只不过这里记的不是传统 SLAM 那种显式地图，而是 transformer token 形式的潜在 3D 状态。和论文的 persistent-state 设计是对齐的。

训练：论文写到他们总共用了 32 个数据集，覆盖真实/合成、室内/室外、静态/动态、场景级/物体级等不同设置；训练采用 curriculum，先在 4 视图、以静态数据为主的阶段训练，再加入动态场景和部分标注数据，之后提升分辨率，最后冻结 encoder，只训练 decoder 和 heads，并把序列长度扩展到 4 到 64 视图，专门强化长上下文建模。实现上，图像 encoder 用的是 ViT-Large，并从 DUSt3R 的 encoder 权重初始化；decoder 是 ViT-Base；state 大小是 768 个 token，每个 768 维。

loss*：主要是几何监督。采用了类似 MASt3R 的 confidence-aware 3D regression loss 去监督 pointmap；位姿则用 quaternion 和 translation 的 L2 损失；如果输入是 raymap，还会再加一个 RGB 重建损失去约束查询视角下的颜色预测。对于有 metric 3D 标注的数据，它直接学 metric-scale pointmap；对尺度未知的数据，则关闭 raymap querying，避免监督尺度和状态内部尺度不一致。也就是说，这篇文章不是纯生成式 3D，而是一个以几何回归为核心、再叠加 persistent memory 机制的系统。

实验结果：在视频深度估计上，CUT3R 在 Sintel、Bonn、KITTI 上整体优于 DUSt3R 和 MASt3R，并且作为在线方法，相比依赖全局对齐的方案速度非常快；论文表里在 KITTI 的设定下给出 16.58 FPS，并指出相较 MonST3R 的相关设定接近 50× speedup。在相机位姿估计上，论文明确写它在几组数据集上取得了 online methods 中最好的整体表现。在 7-Scenes 和 NRGBD 的 3D reconstruction 对比里，它也能在保持在线处理的同时，达到接近甚至超过某些离线全局对齐方法的效果。

state update 的意义：作者把正常在线更新和一种“revisit”设置做比较，后者相当于在更新时带入更强的全局上下文，结果重建会更好，尤其是难区域。这说明 persistent state 真正在承担“把跨时刻上下文聚合起来”的作用。论文后面还展示了通过 state readout 推断未见结构的定性结果，比如仅输入单张图，再查询另一个视角时，模型可以推断出输入里没直接看见的地面、灌木、家具等结构，虽然高频细节可能不足，但大体几何和视角关系是对的。

限制：像许多在线方法一样，长序列会漂移，因为没有显式全局对齐；其次，它对未见区域的推断本质上还是 deterministic regression，所以如果查询视角离已观测视角太远，结果会偏模糊；再有就是 recurrent 训练本身比较耗时。它还没有完全解决 SLAM 里的长期一致性问题，也还没真正做到高保真生成式补全。

LoGeR ​

核心

把原本只擅长短窗口的前馈几何模型，真正拉到分钟级长视频、公里级轨迹上，还尽量不依赖传统 SLAM/BA 那种后端优化。

motivation：一个是 context wall，也就是全局双向注意力的二次复杂度让长序列根本吃不下；另一个是 data wall，也就是模型大多只在几十到一百多帧的“短泡泡”数据上训练，即便推理时做高效化，也很难泛化到大尺度长轨迹。

想法概括：chunk-wise processing + hybrid memory。 把长视频切成一段一段的小块来处理，每个 chunk 内部仍然用强的双向几何 backbone 做高质量局部推理，但 chunk 和 chunk 之间不再靠单一记忆机制传信息，而是分成两条路：一条是SWA（Sliding Window Attention）， ，负责相邻 chunk 之间的“无损、精细、短程对齐”；另一条是 TTT（Test-Time Training）fast weights，负责跨很多 chunk 的“压缩、长期、全局锚定”。作者称为 hybrid memory，因为他们认为长程几何一致性和相邻边界精细拼接，本来就不是一种 memory 能同时做好的。

LoGeR 单个 block 的流动：先做 per-frame attention，提取每帧内部 2D 视觉特征；然后插入 SWA，让当前 chunk 和前一个 chunk 的边界特征直接通信；接着做 chunk-wise TTT，用历史 fast weights 把长期信息注入当前 chunk，并在处理完当前 chunk 后再更新这组 fast weights；最后在当前 chunk 内做一次强的 chunk-wise bidirectional attention，完成真正高质量的局部几何推理，最后接轻量 decoder 输出 pointmap 和 camera pose。SWA 只在部分层里稀疏插入，而 TTT 则承担长期全局传播。

SWA 和 TTT 在 LoGeR 里扮演的是两个时间尺度完全不同的角色。 SWA 是“局部记忆”，只看前后相邻 chunk，所以它保留的是未压缩的高精度几何 token，非常适合解决 chunk 边界拼接时的细节连续性问题；TTT 是“全局记忆”，它把历史信息压进一组 fast weights 里，理论上感受野可以无限长，更适合传递场景的粗几何、全局尺度、整体坐标系锚定。只靠 TTT 这种压缩记忆，会损失 dense 3D 重建最需要的高精度局部信息；只靠局部拼接，又会在很长序列上发生 scale drift。

其中 TTT 这部分不是普通 RNN hidden state，而是“apply-then-update”的 fast-weight memory。当前 chunk 进来时，先用历史 fast weights 去调制当前 token 表示；处理完后，再用当前 chunk 的信息更新 fast weights，给后续 chunk 用。作者实现里把 fast-weight module 做成了 SwiGLU MLP，并使用 Muon 做 test-time 更新；他们强调这种记忆本质上是有损压缩，适合长期信息，但容量并不是无限的，所以当序列远超训练上下文时，仍然会有漂移风险。

因此，LoGeR 其实还有一个很现实的工程判断：它并没有声称“只靠一个 elegant memory 就彻底解决无限长视频”。论文明确承认，TTT 的有效范围仍受训练上下文长度限制；当序列长到非常夸张时，作者会在推理时使用 periodic state resets，并提供一个 LoGeR* 变体，用纯前馈的 pose alignment 把 chunk 进一步对齐到统一全局坐标系，缓解长期误差积累。也就是说，它不是那种“理论上无限长所以实践上万无一失”的工作，而是一个相当务实的折中方案。

仅仅把注意力做高效是不够的，所以他们构造了一个偏重大尺度场景的训练混合数据，并用了 curriculum。训练上先从 48 帧 / 4 chunks 开始，再增加 chunk 密度到 12 chunks，最后把总上下文扩到 128 帧 / 20 chunks；同时数据上用了 14 个 real + synthetic 数据源，包括 ARKitScenes、DL3DV、MegaDepth、ScanNet、ScanNet++、Spring、TartanAir、TartanAirV2、Waymo 等，并特别加重了像 TartanAirV2 这类更能提供长程几何信号的大场景数据。作者还给出训练成本：总共约 40k steps，先在 32×H100 上训两天，再在 32×H200 上训两天。

实验：长序列上，在 KITTI 的 ATE 指标里，LoGeR* 把平均误差做到 18.65 m，而 TTT3R 是 72.86 m，CUT3R 是 91.62 m，作者自己的 chunk baseline Pi3-Chunk 也还有 52.07 m；LoGeR 不带额外 feedforward alignment 时是 25.44 m。作者因此说相对先前前馈方法在 KITTI 上有 74% 以上的误差下降，这个说法和表里的数字是一致的。

VBR 长程基准：这个数据集里的 7 条序列长达 8,815 到 18,846 帧，轨迹长度 1.4 km 到 11.5 km。在这个 benchmark 上，LoGeR 的平均 ATE 是 5.40 m，LoGeR* 是 5.27 m，明显优于 CUT3R 的 7.75 m、TTT3R 的 7.62 m 和 InfiniteVGGT 的 9.36 m；而且论文和项目页都强调，这些结果是在没有 backend optimization 的前馈设置下得到的。

短序列和室内数据：论文在 7-Scenes、ScanNetV2、TUM-Dynamics 上都做了评测。项目页总结说，在他们的设置下，LoGeR 在 ScanNet 和 TUM 上分别有 80.0% 和 66.1% 的相对增益；附录图还提到在 7-Scenes 的 1k 帧评测里，相对 TTT3R 和 VGG-T³ 的误差下降分别达到 90.3% 和 72.1%，同时比 VGGT 快得多。

消融：作者把 hybrid memory 拆开后发现，无论去掉 TTT 还是去掉 SWA，性能都会下降。在 ScanNet 1000 帧上，LoGeR 的 ATE 是 0.107；去掉 TTT 变成 0.162，去掉 SWA 变成 0.143。在 TUM 1000 帧上，对应是 0.050 / 0.079 / 0.053。这说明作者的主张并不是“TTT 很强”或者“SWA 很强”单独成立，而是两者的互补性才是关键。

效率：在单张 A100 40GB 上跑 500 帧序列时，chunk size 为 64 / 48 / 32 时，速度分别约 9.3 / 10.6 / 12.1 FPS，峰值显存分别约 27.2 / 22.3 / 18.1 GB。作者还提到 SWA 做了 KV-cache 来避免重复计算，所以这篇文章虽然不是常数内存那一类极致 streaming 模型，但已经把“长序列 + 强局部双向推理”的成本控制到了可用范围。

VGGT / π³ 这类模型强在短窗口里的高质量双向几何推理，但上下文上不去；CUT3R / TTT3R / StreamVGGT 这类线性或流式模型强在长序列可扩展，但容易因压缩记忆而损失边界几何精度或发生长期漂移；LoGeR 则是在两者之间取了一个很聪明的折中：chunk 内仍保留强 bidirectional prior，chunk 间再用 hybrid memory 接起来。从研究品味上看，这篇文章的价值不只是又刷了一个榜，而是提出了一个很有启发性的观点：长程几何重建不是“选 full attention 还是选 recurrent memory”的二选一问题，而是要把不同时间尺度的记忆机制分层设计。

边界：TTT memory 的有效长度仍受训练长度约束，所以超长序列上还需要 periodic resets，LoGeR* 还额外引入了 feedforward alignment；这说明它离“真正无漂移、无限长、纯前馈”的理想状态还有距离。
