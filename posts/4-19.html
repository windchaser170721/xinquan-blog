DA3 ​

核心

用一个尽量简单的 Transformer，统一处理单目深度、多视图几何、相机位姿估计，甚至进一步接到 3D Gaussian 表示上。从任意数量视角恢复 visual space”，而且输入图像可以有相机位姿，也可以没有。

它在方法论上和 VGGT 那类“多头、多任务、专门几何结构”的路线有点不一样。DA3 主张一个 plain transformer，不一定需要很复杂的专门几何架构；同时，它不想再显式预测一大堆互相关联的几何量，而是用一个统一的 depth-ray representation 来做建模。

给你 1 张、2 张、很多张图，甚至一段视频，模型能不能都用同一套骨干网络来理解“这个空间长什么样”。DA3 给出的答案是可以。同一个 any-view 主系列模型，只要换输入配置，就能做单目深度估计、多视图一致深度、pose-conditioned depth、相机位姿估计，甚至还能做 3D Gaussian estimation。这和过去“单目模型一套、多视图模型一套、位姿估计再一套”的分裂式设计很不一样。

输出：强调空间一致性，它会输出一致的 depth 和 ray maps，从而在不同视角之间形成可以融合的几何结构；DA3 不只是“做深度”，而是在做一个更完整的 visual geometry foundation model。Depth Anything 2 更偏强单目深度，而 DA3 把任务扩展到了 any-view geometry。DA3 不仅在多视图深度和相机位姿上超过了 VGGT，而且在单目深度上也超过了 DA2。它想用一套统一模型把“从单目到多视图”的几何能力都包进去。

性能：DA3 在它们新建的 visual geometry benchmark 上，相比此前 SOTA 的 VGGT，平均提升了 35.7% 的相机位姿精度 和 23.6% 的几何精度；arXiv 摘要里给出的数字更高，写的是 44.3% pose accuracy 和 25.1% geometric accuracy。这两个数字不完全一致，通常可以理解为项目页与论文版本在评测汇总或版本更新上有差异，但共同结论是一致的：DA3 在发布时把 VGGT 压过去了。

延伸点：第一，官方明确提供了 DA3-Streaming，说可以用滑窗式 streaming inference 处理超长视频，并把显存压到 12GB 以下。这说明 DA3 不只是静态多视图模型，也在往长序列视频几何建模方向扩。第二，它原生支持 3D Gaussian estimation，官方 repo 和模型卡都把这一项列成主能力；项目页还专门展示了“冻结 backbone、再训练一个 DPT head 来预测 3DGS 参数”的 feed-forward 3D Gaussians 结果。

对比：VGGT 更像“统一预测 camera / depth / point map / tracking feature 的几何大模型”，而 DA3 更强调“统一的 depth-ray 目标 + plain transformer 足以把这些空间关系学出来”。前者更像通过多种显式几何输出联合约束空间结构，后者更像在表示层面做了极简化，试图用更少的任务头、更统一的目标去覆盖更广的视角输入。DA3 的方法哲学比 VGGT 更“minimalist”。

应用：2026 年的一篇工作直接把 DA3 的 feature space 拿来当扩散模型的 latent space，以引入跨视角几何先验；还有一篇做任务感知观测接口的机器人论文，把 Depth Anything 3 用作几何通道，为策略提供更稳定的语义-几何观测；另一篇最近的测试时自监督校正工作也 把 DA3 和 VGGT 都作为可提升的基础几何模型。

DA3 是把 Depth Anything 从“强单目深度模型”推进成“任意视角 3D 几何基础模型”的版本。它提出：3D 几何大模型不一定要靠复杂专用结构堆出来，一个 plain transformer 加统一 depth-ray 表示，也能把单目、多视图、位姿和 3DGS 这几件事较优雅地统一起来。

详细展开昨天说的第一点 ​

3D 几何到底该在策略里从哪里进、以什么频率进、用什么表示进。几何信息从哪里来、注入到 policy 的哪一层、解决的主要瓶颈是什么、代价是什么。VLA 里真正缺的到底是什么，3D 几何应该放在模型的哪个位置，才能既提升操作，又不破坏原有的语言—视觉—动作能力。这几篇论文分别代表了六种不同的答案：把几何放在输入表征里、放在动作离散化里、放在动作专家中间层里、放在视觉骨干里、放在动作头里，或者干脆放在外部慢系统 + 内部快系统的异步接口里。

SpatialVLA ​

VLA 本身缺空间感。 它做了两件基础的事。第一件是 Ego3D Position Encoding，把 3D 信息注入输入观察；第二件是 Adaptive Action Grids，把机器人动作改写成自适应离散的空间动作网格，而不是只把动作看成普通连续控制量。它还在 110 万条真实机器人轨迹上做了预训练，并把 action grids 设计成能在新机器人或新设置上重新离散化，用于跨平台迁移。SpatialVLA 的真正创新点是把空间表征和空间动作表征同时提升到基础建模对象，如果动作发生在 3D 世界里，那么 observation token 和 action token 都应该带有空间结构。

SpatialVLA 的思想：既然操作本质是空间问题，那就从最上游重写输入和动作的表示。这种思路的优点是很干净，因为它不需要依赖一个外部几何 foundation model，也不把几何当成外挂模块；空间性直接刻进了 tokenization。缺点也来自这里：一旦你从输入和动作编码层面重写表示，模型的整个训练和迁移流程都会比较重，而且它更像一个“从头定义 spatial-aware VLA”的方案，不太像给现有 OpenVLA 一类模型直接打补丁。它告诉你空间结构必须进入 VLA，但还没有把“如何最省成本、最少破坏地注入 3D”做到极致。

PointVLA ​

怎么给已经训练好的 2D VLA 无痛补上 3D 世界知识。 出发点：大规模 VLA 已经吃了海量 2D 预训练数据，你不可能每次想加 3D，就把整套 backbone 重训一遍，于是提出一个很工程的方法：冻结原始 action expert，只通过轻量 modular block 把 point cloud 特征注进去；为了不破坏已有表征，它还做了 skip-block analysis，专门找出 action expert 里 相对没那么关键 的 block，把 3D 信息注到这些位置。它报告的几个现象也很说明问题，比如四任务各 20 个 demonstration 的 few-shot multi-tasking、区分真实物体与照片、以及对训练中没见过的桌面高度的适应。

PointVLA 的思想：本质上是一个参数干预理论：不是所有层都值得接 3D，甚至大部分层都不该接。它隐含的观点其实很强——原有 2D VLA 并不是什么都缺，它缺的是在动作生成阶段可用的空间判别力。 所以 PointVLA 选择不碰视觉—语言主干，而是精准地干预 action expert。3D 是一种高价值但高扰动的信号，应该只在最需要的地方进入。

GeoAware-VLA ​

很多 VLA 在新相机视角下失效，不是因为动作学不会，而是因为从 2D 图像里学 3D 一致性太难，视觉编码器本身几何敏感度不够。 它直接换掉或增强视觉骨干的几何能力：用一个冻结的、预训练好的几何视觉模型做 feature extractor，再加一个轻量 projection 层把这些 feature 适配给 policy decoder。

GeoAware-VLA 的思想：它把 3D 问题重新表述成 viewpoint invariance 问题。很多 manipulation 失败表面上看像 grasp 错了、move 错了，实际上根源是视觉表征对视角变化不稳。有时候不需要在 policy 末端拼命补几何，只要前端视觉骨干先把几何一致性学好，后面策略自然轻松很多。它和 PointVLA 形成了很漂亮的对照：PointVLA 认为 3D 应该进 action expert，GeoAware-VLA 认为 3D 应该进 vision backbone。3D 接口 是这个方向的核心变量。

FALCON ​

于是它给出的答案是：把 rich 3D spatial tokens 直接送到 action head，而不是拼到 vision-language backbone。 前面几类方法有两个问题：一种依赖专门传感器，跨模态转移差；另一种把弱几何 cue 粗暴拼进 backbone，结果反而伤到 vision-language alignment。它利用 spatial foundation models 从 RGB 中抽取几何先验，又设计了 Embodied Spatial Model，在有需要时还能融合 depth 或 pose；这些空间 token 最终由一个专门的 Spatial-Enhanced Action Head 消化，而不是与原始语言视觉 token 无差别混合。论文在 3 个仿真 benchmark 和 11 个真实任务上报告了强结果，并强调在 clutter、spatial-prompt conditioning、物体尺度和高度变化下都更稳。

FALCON 的思想：语言推理和几何推理虽然都重要，但它们不一定要在同一个 token 空间里无差别融合。很多早期多模态工作默认 统一 token space = 更强融合 ，但 FALCON 其实反过来说，VLA 里空间信息最该服务的是 action grounding，而不是去扰动原本已经很强的语言—视觉对齐。因此它把几何信息延后到 action head 再发挥作用。这种结构性克制和 backbone 早融合不同，让几何服务动作，而不是让几何接管全部表征。

3D-Mix ​

它做了一个九种 VGGT 融合方案的 pilot study，发现最好的是 semantic-conditioned gated fusion ——也就是根据任务上下文，自适应地平衡 2D 语义特征和 3D 几何特征。它提出 3D-Mix 这个 plug-and-play 模块，能兼容 GR00T-style 和 π-style VLA。

3D-Mix 的思想：除了 gated fusion，它思考3D 语义混合应不应该受任务上下文控制。3D cue 不是每时每刻都同样重要：抓取前接近物体时，几何和尺度很重要；高层语义决策时，语言任务意图更重要。3D-Mix 承认：2D 语义和 3D 几何不是静态加权关系，而是动态竞争关系。谁该主导，应该由当前任务上下文决定。这个思想可以扩展成长期项目，可以继续追问：这个 gating 能否由 contact stage、subgoal phase、uncertainty 或 scene dynamics 来控制，而不只是由静态 token 交互来学。

R3DP ​

可以很聪明地接入 3D foundation model，但闭环控制不能等它慢吞吞推理。 R3DP 的关注点是 几何信息该以什么时间频率进入系统。 它提出一个 asynchronous fast-slow collaboration 结构：重的 3D 模型只在稀疏关键帧上运行，作为 slow system；中间帧则由轻量的 Temporal Feature Prediction Network, TFPNet 预测几何特征，以维持实时控制；同时它还引入 MVFF，在多视角情况下显式使用相机内外参来做 feature 融合。相比单视角和多视角 diffusion policy 基线，平均成功率分别提升 32.9% 和 51.4%；相较 naive 的 DP+VGGT 直接拼接，它的推理时间降低 44.8%。

R3DP 的思想：它把 3D 接口问题 从网络结构继续推进到了系统时序结构。不可能指望一个大几何模型每 30Hz 都完整跑一遍，把大模型变成稀疏校正器，把小模型变成连续补全器。

演化脉络 ​

SpatialVLA 关心的是把空间性刻进输入和动作；PointVLA 关心的是最小代价地把 3D 插入现有 VLA 的 action expert；GeoAware-VLA 关心的是从视觉骨干层面补足几何一致性；FALCON 关心的是让几何在 action head 而不是 backbone 发挥作用；3D-Mix 开始系统比较并提出按上下文动态控制 2D/3D 融合；R3DP 则进一步把问题推进到实时性和异步更新。也就是说，这条线并不是重复劳动，而是在不断逼近一个更完整的问题：几何先验应该以什么表示、在什么层级、以什么频率，进入机器人动作决策。

idea ​

第一，3D 不是普通附加模态，而是结构化约束。这些工作越往后，越少有人把 3D 当“再多一组 feature”。不管是 GeoAware-VLA 的 viewpoint invariance，FALCON 的 spatial-to-action，还是 3D-Mix 的 gated fusion，本质上都在说明：几何的作用是给策略一个关于物体位置、尺度、视角一致性、空间关系的结构约束，而不是多一个传感器而已。

第二，几何最该服务的是动作 grounding，而不是全局表征统一。这点在 PointVLA 和 FALCON 身上体现得最明显。PointVLA 有意识地冻结原 action expert，再把 3D 注入少数 block；FALCON 更明确地把 spatial tokens 交给 Spatial-Enhanced Action Head。这说明当前更有效的范式，往往不是让 3D 从头接管视觉语言主干，而是让它在动作决策的最后几步里提供关键判别力。

第三，3D 接入不是一次性设计，而是一个层级化、时序化接口问题。SpatialVLA 讨论输入和动作空间，GeoAware-VLA 讨论视觉编码器，FALCON 讨论动作头，R3DP 讨论异步频率；这些问题叠在一起，才构成一个真正可部署的 geometry-conditioned VLA。换句话说，未来最强的方法很可能不是“某一层加 3D 最优”，而是一个多层接口协同方案：前端少量几何对齐，中段上下文控制融合，末端动作头强化，再加异步慢系统定期校正。这个推断是我基于这几篇的共同趋势做的归纳。

如果再往“你自己的项目还能怎么做”这一步走，我会说，最值得继续推的已经不是“再找一个新的 3D feature source”，而是研究 几何先验的调度机制。比如说，哪些时刻该更多信 3D，哪些时刻该更多信语义；几何 token 是否应该随 task phase 改变注入深度；慢系统的几何校正是否可以只在 uncertainty 高的时候触发；scene flow、point flow、depth token、camera-aware feature 这些不同表征，是否应该在不同动作阶段扮演不同角色。前面这些论文已经把“3D 有用”证明得差不多了，下一步真正有原创空间的，是3D 如何被按需调用、按层调用、按阶段调用。这一点我觉得就是你能把项目做大的地方。

如果你愿意，我下一条可以继续把这六篇压成一张更实用的“研究地图”：我直接帮你总结成 三个可做的原创题目方向，每个方向都写清楚适合借鉴哪几篇、最合适的 baseline、核心创新点该落在哪。

思考 ​

早在 pointvla 中就提到过 3d 数据规模远小于 2d 机器人数据，硬改肯定会破坏学会的2d表征，pointvla 做了一个 skip-block analysis：在 DexVLA 的 32 个 diffusion transformer blocks 里，逐个或连续跳过某些 block，看性能掉不掉。发现前 11 个 block 很关键，跳了会明显坏掉；但从第 11 个以后，部分 block 在推理阶段被跳过，性能影响没那么大，甚至最多可以连续跳过 5 个 block 才彻底失败。于是他们就把 3D 特征选择性地注入到这些“相对没那么关键”的 block 里。这个设计把点云当作对后段动作生成过程的几何修正。
